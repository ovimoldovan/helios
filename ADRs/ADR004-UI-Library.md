# ADR003-UI Library choice

## Status
Accepted

## Context
We needed a UI Library to make our lives easier when building the frontend

## Decision
We chose shadcn as it has a large set of components and it was easy to install. Also, it has a designer & a plugin, making our lives even easier.

## Consequences
Pros: large set of components; easy to install; has a designer & a plugin; easy to customize; theme defined per project

Cons: Visual Studio (code) users don't benefit of the IDE plugin thus having to manually install components and preview them in the browser

## Alternatives Considered
Mantine UI: rejected because it was hard to customize using TailwindCSS
